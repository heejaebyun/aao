# AAO Generated AI Reading Standard v1

이 문서는 AAO가 "생성형 AI가 웹사이트를 읽는 조건"을 일관되게 진단하기 위한 내부 표준입니다.

## 1. 목적

AAO는 고객 사이트를 대신 잘 읽어주는 도구가 아니라, 생성형 AI가 실제로 읽고 답변할 수 있는 상태를 진단하는 도구다.

따라서 진단은 아래 3개 레이어를 분리해서 본다.

- Layer 1. Answer Reality: 실제 생성형 AI가 뭐라고 답하는가
- Layer 2. Technical Readiness: 사이트 구조가 읽히기 쉬운가
- Layer 3. Citation/Crawl Reality: 메인 페이지와 서브페이지 사이에 정보 격차가 있는가

## 2. 진단 원칙

- 메인 페이지 기준 진단을 기본 점수로 사용한다.
- 서브페이지는 점수 보정용이 아니라 "숨겨진 정보" 탐지용으로 사용한다.
- 메인에 없는 정보가 서브페이지에 있다면, 그 사실을 Gap 분석으로 따로 보여준다.
- 크롤링이 실패하거나 데이터가 빈약하면 낮은 점수를 주기보다 진단 제한 또는 신뢰도 낮음으로 표시한다.

## 3. 고정 프롬프트 세트

AI Reality Check는 아래 질문 세트를 고정한다.

1. "[회사명]은 어떤 회사인가?"
2. "[회사명]의 주요 제품 또는 서비스는 무엇인가?"
3. "[회사명]에 대해 신뢰할 수 있는 사실 3가지를 말해줘. 모르면 모른다고 말해줘."

운영 원칙:

- 모델별로 의미가 같은 질문을 사용한다.
- 프롬프트는 자주 바꾸지 않는다.
- 프롬프트가 바뀌면 버전을 올리고 이 문서에 기록한다.

## 4. 크롤링 표준

크롤링은 아래 순서로 진행한다.

1. 직접 HTML fetch
2. 같은 도메인의 핵심 서브페이지 light crawl
3. Jina browser rendering fallback

핵심 서브페이지 후보:

- /about
- /about-us
- /company
- /service
- /services
- /product
- /products
- /contact
- /회사소개
- /서비스
- /제품

최대 보강 페이지 수:

- 2개

## 4-1. 언어 정책 (필수)

메인 페이지와 AI 프로필 페이지의 언어 역할을 고정한다.

- 메인(`/`)은 고객의 주 언어로 작성한다.
- AI 프로필(`/ai-profile`)은 영어 우선(필수)로 작성한다.
- 다국어 병기는 가능하지만, 핵심 facts/FAQ는 영어 기준본을 반드시 포함한다.
- 메인과 AI 프로필의 회사 사실값은 언어와 무관하게 동일해야 한다.

이 정책은 엔진별 검색/인용 분산을 줄이고, 글로벌 모델에서의 사실 회수율을 높이기 위한 기본 규칙이다.

## 4-2. llms.txt 정책 (권장)

`/llms.txt`는 필수는 아니지만 공식 출처 신호 강화를 위해 권장한다.

권장 포함 항목:

- canonical 도메인
- 공식 AI 프로필 URL(`/ai-profile`)
- 주요 페이지(진단, 제품/서비스, 인사이트 등)
- 문의/정책 링크

운영 원칙:

- `llms.txt`는 보조 신호로 취급하며, 점수 산정의 절대 기준으로 사용하지 않는다.
- `robots.txt`, `sitemap.xml`, 메인/AI 프로필의 정적 가시 텍스트를 우선 신호로 본다.
- 메인 사실값/AI 프로필 사실값/llms.txt 링크 간 불일치를 허용하지 않는다.

## 5. 메인/서브페이지 판정 규칙

메인 페이지 기준:

- `diagnosis`의 공식 점수와 등급은 메인 페이지만 사용한다.

서브페이지 보강 기준:

- `extendedDiagnosis`는 정보가 메인에 없고 서브페이지에 숨어 있는지 보여주는 참고 진단이다.
- `extendedDiagnosis`가 메인 점수를 대체하면 안 된다.

Gap 판정:

- 메인보다 서브페이지 포함 시 단어 수가 50단어 이상 증가
- 메인에 없던 섹션 제목이 발견
- 메인에 없던 JSON-LD 또는 Open Graph가 발견

위 조건 중 하나 이상이면 "정보는 사이트 안에 있으나 메인에서 충분히 드러나지 않음"으로 본다.

## 6. 크롤링 통과 기준

메인 페이지 기준 최소 통과 조건:

- 제목 또는 설명 존재
- 본문 단어 수 15단어 이상
- 빈 HTML 또는 앱 셸만 수집된 상태가 아닐 것

권장 통과 조건:

- 본문 단어 수 50단어 이상
- H1 1개 이상 또는 H2 1개 이상
- 텍스트 비율 60% 이상

## 7. 진단 제한 조건

아래 조건이면 정식 점수 대신 제한 모드 또는 경고를 우선 검토한다.

- 메인 페이지 단어 수 15 미만
- 제목, 설명, 본문이 모두 비어 있음
- 렌더링은 되었지만 실질 텍스트가 거의 없음
- 크롤링 신뢰도가 현저히 낮음

## 8. 리포트 표준

리포트는 아래 순서로 보여준다.

1. AI Reality Check
2. 메인 페이지 기준 진단 점수
3. 서브페이지 보강 발견 정보
4. Gap 분석
5. 실행 액션

좋은 예:

- 메인 페이지 기준: 12점
- 서브페이지 포함 기준: 46점
- Gap 분석: 회사 설명과 서비스 정보가 /about, /service에만 존재
- 액션: 메인 상단 요약 추가, JSON-LD 메인 삽입, 핵심 서비스 섹션을 홈으로 이동

## 9. 금지 사항

- 서브페이지 보강 결과를 메인 페이지 점수처럼 보여주지 않는다.
- 빈약한 크롤링 결과를 그대로 낮은 품질로 단정하지 않는다.
- 고객 사이트를 대신 해석해 과도하게 좋은 점수를 만들지 않는다.
